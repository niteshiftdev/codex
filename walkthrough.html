Quickstart

Up & Running in 3 Steps

From install to your first AI-powered turn in under a minute.

Install the SDK

Add the package to your project. The SDK bundles pre-compiled Codex binaries for all major platforms.

terminal 
npm install @openai/codex

Create a Codex Client & Thread

Instantiate the client, then start a thread. Each thread is a persistent conversation session the agent remembers across turns.

agent.ts 
import { Codex } from "@openai/codex";

// Create the client — picks up CODEX_API_KEY from env
const codex = new Codex();

// Start a new conversation thread
const thread = codex.startThread({
  model: "o4-mini",
  workingDirectory: "./my-project",
  sandboxMode: "workspace-write",
});

Run a Turn

Send a natural-language prompt. The agent reads your code, executes commands, patches files, and returns a final response.

agent.ts (continued) 
const turn = await thread.run(
  "Add input validation to the signup form"
);

// The agent's natural-language summary
console.log(turn.finalResponse);

// Every action it took: commands, file edits, reasoning
for (const item of turn.items) {
  console.log(item.type, item);
}

// Token usage
console.log(`Tokens: ${turn.usage?.input_tokens} in, ${turn.usage?.output_tokens} out`);
Streaming

Real-Time Event Stream

Use runStreamed() to get live updates as the agent thinks, executes, and edits. Perfect for building reactive UIs.

streaming.ts 
const { events } = await thread.runStreamed(
  "Refactor the auth module to use JWT"
);

for await (const event of events) {
  switch (event.type) {
    case "thread.started":
      console.log(`Session: ${event.thread_id}`);
      break;

    case "item.started":
      console.log(`▶ ${event.item.type} started`);
      break;

    case "item.completed":
      if (event.item.type === "command_execution") {
        console.log(`$ ${event.item.command} → exit ${event.item.exit_code}`);
      }
      if (event.item.type === "file_change") {
        console.log(`✎ Patched ${event.item.changes.length} file(s)`);
      }
      break;

    case "turn.completed":
      console.log(`✓ Done — ${event.usage.output_tokens} tokens`);
      break;
  }
}

Event Lifecycle

thread.started

Session created — save thread_id to resume later

turn.started

Agent begins processing your prompt

item.started → item.updated → item.completed

Each action (reasoning, command, file edit, MCP call) follows this lifecycle

turn.completed

All work done — usage stats available

turn.failed

Error occurred — check event.error.message

Capabilities

Everything the Agent Can Do

Each action the agent takes is represented as a typed item in the turn results.

💬

Agent Messages

Natural-language responses explaining what the agent did and why. Returned as agent_message items.

💻

Command Execution

Runs shell commands in a sandboxed environment. Captures stdout, stderr, and exit codes.

📝

File Changes

Patches files with precise diffs — adds, updates, and deletes. Each change tracked by path and kind.

🧠

Reasoning

Extended chain-of-thought traces so you can see how the agent arrived at its plan.

🔌

MCP Tool Calls

Invoke external tools via Model Context Protocol servers — databases, APIs, custom integrations.

🌐

Web Search

Agent can search the web for documentation, APIs, and solutions when enabled.

✅

Todo Lists

Agent maintains a running task list as it works through complex, multi-step problems.

📦

Advanced

Structured Output

Constrain the agent's final response to match a JSON Schema. Great for building pipelines that consume typed data.

structured.ts 
const schema = {
  type: "object",
  properties: {
    bugs: {
      type: "array",
      items: {
        type: "object",
        properties: {
          file:     { type: "string" },
          line:     { type: "number" },
          severity: { type: "string", enum: ["low", "medium", "high"] },
          summary:  { type: "string" },
        },
        required: ["file", "line", "severity", "summary"],
      },
    },
  },
  required: ["bugs"],
};

const turn = await thread.run("Audit src/ for bugs", {
  outputSchema: schema,
});

const report = JSON.parse(turn.finalResponse);
console.log(`Found ${report.bugs.length} bugs`);

// Save the thread ID after first turn

  model: "o4-mini",
  workingDirectory: "./my-project",
});

Security

Sandbox Modes

Control exactly what the agent can access. Choose the right level for your use case.

🔒

read-only

Agent can read the entire filesystem but cannot write anywhere or access the network. Safest option for analysis tasks.

📁

workspace-write

Reads anywhere, writes only to the working directory and temp dirs. The recommended default for most coding tasks.

⚠️

danger-full-access

No restrictions. Use only in fully trusted environments like CI pipelines or ephemeral containers.
